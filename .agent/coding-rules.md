# コーディング規約とルール

## コード編集時の差分確認ルール

大規模なファイル書き換え（`write_to_file`でOverwrite=true）を行った際、**特にコード量が減少した場合**は必ず以下を実施すること：

### 必須手順

1. **差分確認**
   ```bash
   git diff HEAD~1 <変更したファイル>
   ```

2. **削除内容の検証**
   - 削除された機能がないか確認
   - 意図しない削除がないか確認
   - コメントや重要なロジックが失われていないか確認

3. **ユーザーへの報告**
   - 削除された行数と追加された行数
   - 削除された主要な機能のサマリー
   - 変更が意図通りであることの確認

### 理由

- 大規模な書き換えでは、予期しない機能削除が発生しやすい
- コード量の減少は、重複削除だけでなく機能削除の可能性もある
- 事前確認により、バグや機能喪失を防止できる

### 例外

以下の場合は、この確認を省略してもよい：
- コメントのみの削除
- 明らかな重複コードの削除
- リファクタリングによる簡潔化（ロジックは保持）
